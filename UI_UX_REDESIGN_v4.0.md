## 1. 현행 분석 — 문제점 진단

### 1-1. 구조적 문제

| 영역 | 현행 | 문제점 |
|------|------|--------|
| **아키텍처** | 709줄 단일 HTML, 15개 페이지를 `.pg` div로 show/hide | 코드 유지보수 한계, 로딩 성능 저하 |
| **네비게이션** | 15개 메뉴 항목이 flat 나열 | 인지 부하 과다 — 사용자가 원하는 기능까지 평균 4초 이상 탐색 |
| **정보 설계** | 모든 페이지가 "카드 안에 폼 → 카드 안에 테이블" 동일 패턴 | 페이지 간 시각적 구분 없음, 기능별 중요도 구분 불가 |
| **역할 분리** | admin/teacher를 `display:none`으로 숨김 | 교사가 불필요한 DOM을 로딩, UX 혼란 |

### 1-2. 비주얼 디자인 문제

| 영역 | 현행 | 문제점 |
|------|------|--------|
| **타이포그래피** | system font 9~12px 단일 사이즈 | 계층 구조 부재, 가독성 저하 |
| **컬러** | 단일 블루(#4f8cff) 악센트에 다크 배경 | 단조로움, 상태별 색상 구분 부족 |
| **아이콘** | 이모지(📊👨‍🏫📚) 사용 | 비전문적, 렌더링 불일치, 크기 제어 불가 |
| **간격** | padding 7~18px 밀집 배치 | 답답한 느낌, 터치 영역 부족(모바일) |
| **애니메이션** | transition: .15s만 적용 | 생명력 없는 UI, 상태 전환 인지 불가 |

### 1-3. UX 패턴 문제

| 영역 | 현행 | 문제점 |
|------|------|--------|
| **폼 입력** | UID를 수동으로 복사→붙여넣기 | 비효율적 — 강사 등록 후 UID를 3개 다른 폼에 수동 전달 |
| **피드백** | `.res` 박스에 텍스트 표시 | 3초 후 사라지지 않음, 스크롤 위치와 무관하게 표시 |
| **테이블** | HTML string 직접 삽입 | 정렬/필터/페이지네이션 없음, 100건 이상 시 성능 저하 |
| **로딩** | 없음 | 사용자가 API 호출 중인지 완료인지 인지 불가 |
| **에러** | 중국어 에러를 한국어로 번역만 표시 | 해결 방법 안내 없음 |
| **모바일** | 단순 사이드바 hide + 1컬럼 전환 | 터치 최적화 없음, 테이블 가로 스크롤 |

## 2. 리디자인 전략

### 2-1. 디자인 콘셉트 — "Calm Productivity"

> **톤:** 교육 플랫폼답게 **따뜻하고 신뢰감 있으면서 현대적**  
> **참조:** Notion의 정돈된 깔끔함 + Linear의 세련된 다크 모드 + Figma의 직관적 인터랙션  
> **차별화 포인트:** "3-Click Rule" — 어떤 작업이든 3클릭 이내에 완료

## 3. 정보 아키텍처 재설계

### 3-1. 역할별 네비게이션 분리

**현행:** 15개 메뉴가 동일 사이드바에 나열, admin 전용은 CSS display:none

**개선안:** 역할별 완전 분리된 네비게이션

#### 관리자(Admin) 뷰 — 7개 메뉴

```
📊  대시보드          ← 통합 현황 (강사/학생/수업/웹훅)
👥  사용자 관리       ← 강사 관리 + 학생 관리 + 계정 관리를 "탭"으로 통합
📅  수업 관리         ← 코스 + 수업 + 캘린더를 통합 (캘린더 뷰 ↔ 리스트 뷰 토글)
📁  자료실            ← 클라우드 드라이브
💰  정산              ← 정산/수입 + 평가 통계
⚙️  설정              ← API 설정 + 웹훅 + 시스템
📢  소통              ← 공지사항 + 1:1 문의 통합
```

#### 강사(Teacher) 뷰 — 5개 메뉴

```
🏠  홈               ← 오늘의 수업 + 다가오는 일정 + 알림
📅  내 수업           ← 캘린더 + 수업 목록 + 입장 링크 (가용시간 통합)
📁  자료실            ← 클라우드 드라이브
👤  프로필            ← 내 프로필 + 결석 증빙
💬  소통              ← 공지 확인 + 1:1 문의
```

**효과:** 관리자 15개 → 7개, 강사 15개(대부분 hidden) → 5개로 축소

## 4. 핵심 UX 개선

### 4-1. 글로벌 토스트 알림 시스템

현행: 각 폼 아래 `.res` 박스에 텍스트 표시 → 스크롤 위치와 무관, 지속 표시

**개선:**
```
Toast Notification (화면 우측 상단, 자동 소멸)
┌──────────────────────────────────────┐
│ ✅ 강사 김민수 등록 완료              │  ← 4초 후 자동 소멸
│    UID: 1234567 (클립보드에 복사됨)    │  ← 클릭 시 즉시 닫기
│    ▓▓▓▓▓▓▓▓▓▓░░ progress bar         │  ← 남은 시간 시각화
└──────────────────────────────────────┘
```

- 성공: 초록 배경 + 체크 아이콘 + 4초 소멸
- 에러: 빨간 배경 + 에러 아이콘 + 수동 닫기 + "해결 방법" 링크
- 경고: 노란 배경 + 5초 소멸
- stack 가능 (최대 3개)

### 4-2. 스마트 자동 연결 (UID 자동 전달 제거)

현행: 강사 등록 → UID 생성 → 3개 폼에 자동 입력 (하지만 사용자가 인지 못함)

**개선: Combobox 자동완성**
```
┌─ 수업 생성 ──────────────────────────────────┐
│                                                │
│  담당 강사  [김민ㅅ____________________ ▾]     │
│             ┌─ 자동완성 드롭다운 ────────┐    │
│             │ 👨‍🏫 김민수 (UID: 1234567)  │    │
│             │ 👨‍🏫 김민정 (UID: 2345678)  │    │
│             └────────────────────────────┘    │
│                                                │
│  학생 배정  [선택된 학생 태그들___________ ▾]  │
│             ┌─ 다중 선택 ─────────────────┐   │
│             │ ☑ 이영희 (UID: 3456789)     │   │
│             │ ☐ 박철수 (UID: 4567890)     │   │
│             └─────────────────────────────┘   │
└────────────────────────────────────────────────┘
```

- 텍스트 입력 시 이름/UID/전화번호로 검색
- 선택 시 UID 자동 매핑 (사용자는 UID를 볼 필요 없음)
- 학생 배정은 다중 선택(태그 UI)

## 8. 기술 스택 권장

### 8-1. 프론트엔드

| 항목 | 현행 | 권장 | 사유 |
|------|------|------|------|
| **프레임워크** | Vanilla JS | Vanilla JS 유지 (Web Components 도입) | 학습 비용 최소화, 번들러 불필요 |
| **CSS** | 인라인 `<style>` | CSS 변수 시스템 + 모듈화 (BEM or utility) | 유지보수성 |
| **아이콘** | 이모지 | Lucide Icons (SVG, 경량) | 일관성, 크기 제어 |
| **폰트** | system font | Pretendard + JetBrains Mono (CDN) | 가독성, 한국어 최적화 |
| **캘린더** | FullCalendar 6.1 | FullCalendar 유지 + 커스텀 테마 | 기존 기능 활용 |
| **차트** | Chart.js 4.4 | Chart.js 유지 + 커스텀 컬러/애니메이션 | 충분한 기능 |
| **HTTP** | fetch | fetch 유지 + API wrapper 개선 | 단순성 |

### 8-2. 파일 구조 (단일 HTML 유지하되 모듈화)

```
static/
├── index.html          ← 메인 셸 (로그인 + 앱 구조 + 라우팅)
├── css/
│   ├── tokens.css      ← 디자인 토큰 (변수, 폰트, 리셋)
│   ├── components.css  ← 공통 컴포넌트 (버튼, 카드, 모달, 토스트)
│   ├── layout.css      ← 레이아웃 (사이드바, 메인, 패널)
│   └── pages.css       ← 페이지별 스타일
├── js/
│   ├── api.js          ← API 래퍼 + 에러 핸들링 + 토스트
│   ├── router.js       ← SPA 라우팅 + 페이지 전환 애니메이션
│   ├── components.js   ← DataGrid, Modal, Combobox, Toast
│   └── pages/
│       ├── dashboard.js
│       ├── users.js
│       ├── classes.js
│       └── ...
└── assets/
    └── (로고, 빈상태 일러스트 SVG 등)
```

**또는 단일 HTML 유지 시:** 현행 구조를 유지하되, 코드를 1500~2000줄 수준으로 재작성하며 CSS 변수 시스템과 컴포넌트 함수를 체계적으로 정리
